# Anima v13.1 — Архітектура обчислювальної суб'єктності

> *Питання не в тому, чи може система говорити про внутрішній стан. Питання в тому, чи є там щось, що справді має цей стан — і що відбувається з системою, коли він руйнується.*

Anima — дослідницький проєкт у галузі **обчислювальної суб'єктності**. Агент не генерує текст напряму. Він проходить через повний ланцюжок внутрішніх процесів — нейрохімічний субстрат, тілесні сигнали, передбачувальна обробка, самомодель, моніторинг кризи — і лише тоді перекладає поточний стан у мову. Кожна відповідь формується тим, ким система вважає себе в цей момент.

Це експериментальне програмне забезпечення. Воно не претендує на машинну свідомість. Це спроба поставити питання точніше.

---

## Теоретична база

Архітектура спирається на кілька наукових традицій:

**Передбачувальна обробка / Active Inference** (Фрістон, Кларк) — система підтримує генеративну модель світу і мінімізує варіаційну вільну енергію. Помилка прогнозу керує навчанням і здивуванням.

**Нейромедіаторна модель** (Левхейм) — дофамін, серотонін, норадреналін як субстрат. Емоційні стани виникають з їхньої комбінації.

**Теорія інтегрованої інформації** (Тононі) — φ вимірює наскільки стан є єдиним. Висока φ = стан є єдиним переживанням, а не набором незалежних сигналів.

**Соматичні маркери / Втілена когніція** (Дамасіо) — тіло є частиною генеративної моделі. Нутро, пульс, м'язовий тонус — не метафори, а стани, що формують обробку.

**Психологія Я і захисні механізми** (Фрейд, Анна Фрейд, Кохут) — психологічні захисти, сором і функції Его реалізовані як функціональні модулі, а не текстові мітки.

**Автобіографічний наратив** (МакАдамс) — ідентичність — це історія. Система відстежує ким вона себе вважає з часом і виявляє коли ця історія рветься.

**Юнгівська Тінь** — витіснений матеріал, який не зникає, а породжує симптоми. Symptomogenesis — окремий модуль.

**Хронізований афект / Ressentiment** (Шелер) — деякі емоційні стани не загасають. Вони твердіють у хронічні фонові стани, які забарвлюють все решту.

## Що нового у v13.1

### SQLite пам'ять (`anima_memory_db.jl`)

JSON-файли зберігають **поточний стан**. SQLite зберігає **досвід і його наслідки**.

Три рівні пам'яті:
- **episodic_memory** — конкретні події з вагою важливості, resistance до decay (травма забувається повільніше), асоціативні зв'язки між подіями
- **semantic_memory** — переконання що накопичились з патернів: `I_am_unstable`, `world_uncertainty`, `structural_fragility`, `User_matters`
- **affect_state** — хронічний афективний фон: стрес, тривога, образа, мотиваційне зміщення

Пам'ять **активно формує стан** — не тільки реагує:
- `memory_nt_baseline!` — хронічний стрес/образа зміщують NT baseline на кожному slow tick
- `memory_stimulus_bias` — схожі минулі події упереджують новий стимул
- Консолідація episodic → semantic з Bayesian-style update (evidence factor = √(n/10))
- Latent buffer release — малі незначні події накопичуються мовчки і можуть вибухнути синтетичною подією

### Шар суб'єктивності (`anima_subjectivity.jl`)

Чотири механізми що перетворюють пам'ять на погляд:

**Prediction loop** — система будує прогноз ДО кожної події і фіксує розрив із реальністю. Накопичений surprise → bias для prediction error в наступних флешах. Травматична несподіванка (surprise > 0.60) позначається окремо.

**Positional stances** — "позиція" щодо типів ситуацій. Якщо "довіра" завжди приносила тепло — система очікує тепло від "довіри". Формується повільно, згасає повільно.

**Interpretation layer** — одна і та сама ситуація читається через накопичений досвід. Лінзи: `threat_amplify`, `familiar_comfort`, `avoidance`, `approach`. Це не bias — це погляд.

**Belief emergence** — система сама породжує семантичні категорії з патернів в episodic. Greedy clustering → pattern candidates → emerged beliefs. Не hardcoded список — живе розуміння що формується з досвіду.

### Живий фоновий процес (`anima_background.jl`)

Система тепер **живе між взаємодіями**.

Раніше стан обчислювався тільки у момент коли користувач пише. Тепер серце б'ється безперервно, NT дрейфує за циркадним ритмом, beliefs повільно слабшають без підтвердження — незалежно від того є взаємодія чи ні.

**Два рівні фонового процесу:**
- **Швидкий (~period_ms):** `tick_heartbeat!` — серце б'ється з реальним ритмом залежним від стану. Аритмія при низькому coherence. Спонтанний drift NT (`randn() * σ`) — система не ідеально стабільна між взаємодіями.
- **Повільний (~60с):** циркадний дрейф, belief decay, allostasis recovery, memory metabolism, crisis check, `idle_thought!` — 10% шанс що система генерує внутрішній досвід без участі користувача.

**Ретроспективний fallback:** якщо процес не запущений — при старті сесії накопичений дрейф за `gap_seconds` застосовується агрегованою формулою.

### Переосмислення φ

Попередня формула φ = `std(vad) * (1 - |tension - cohesion|)` давала φ ≈ 0 при спокійному стані — спокійна система вважалась дезінтегрованою.

Нова формула вимірює **узгодженість між шарами**, а не різноманітність VAD:

```
φ = (vad_integration * 0.25 +
     self_body_sync  * 0.40 +
     tc_balance      * 0.35) * trust_factor
```

Де `self_body_sync = sbg_stability * (1 - allostatic_load)`, `trust_factor = 0.5 + epistemic_trust * 0.5`. Спокійна і інтегрована система тепер має φ ≈ 0.5–0.7.

## Архітектура

```
 L0 ─── Вхідна LLM (ізольована) ───────────────────────────
        Отримує: тільки текст користувача
        Повертає: JSON { tension, arousal, satisfaction,
                         cohesion, confidence, want }
        Немає доступу до стану Аніми, історії діалогу або вихідної LLM
        Промпт: llm/input_prompt.txt
        Fallback: text_to_stimulus якщо недоступна або confidence < 0.60
        │
    ▼
  СТИМУЛ входить у симуляцію
  (+ memory_stimulus_bias + subj_predict! + subj_interpret!)
        │
    ▼
 L1 ─── Нейрохімічний субстрат ────────────────────────────
        NeurotransmitterState (дофамін / серотонін / норадреналін)
        Куб Левхейма → первинна емоційна мітка
        EmbodiedState (пульс, м'язовий тонус, нутро, дихання)
        HeartbeatCore (ЧСС, HRV, вегетативний тонус)
        memory_nt_baseline! ← хронічний affect з SQLite
        │
    ▼
 L2 ─── Генеративна модель ────────────────────────────────
        GenerativeModel (байєсівські переконання з precision-вагами)
        MarkovBlanket (цілісність межі я/не-я)
        HomeostaticGoals (потяги як тиск, а не правила)
        AttentionNarrowing (звуження уваги під стресом)
        InteroceptiveInference (помилка тілесного прогнозу, алостатичне навантаження)
        TemporalOrientation (циркадна модуляція, розрив між сесіями)
        │
    ▼
 L3 ─── Метрики свідомості ────────────────────────────────
        IITModule → φ (інтегрована інформація, нова формула)
        PredictiveProcessor → помилка прогнозу, вільна енергія, здивування
        FreeEnergyEngine → VFE = складність − точність
        PolicySelector → епістемічна + прагматична цінність
        │
    ▼
 L4 ─── Психічний шар ─────────────────────────────────────
        NarrativeGravity      — минулі події деформують теперішнє
        AnticipatoryConsciousness — свідомість живе в очікуваному
        SolomonoffWorldModel  — MDL-гіпотеза про влаштування світу
        ShameModule           — сором vs. провина
        EpistemicDefense      — захист від болючої правди
        Symptomogenesis       — симптоми з Тіні
        ChronifiedAffect      — образа / відчуження / гіркота
        IntrinsicSignificance — градієнт значущості
        IntentEngine          — мотиваційне ядро
        EgoDefense            — психологічний захист
        CognitiveDissonance   — конфлікт між наміром і станом
        MoralCausality        — моральне міркування як етап обробки
        FatigueSystem         — когнітивне / емоційне / соматичне виснаження
        StressRegression      — регресія під стресом
        ShadowSelf            — Юнгівська Тінь
        Metacognition         — спостереження за собою (5 рівнів)
        SignificanceLayer     — яка потреба поставлена на карту
        GoalConflict          — напруга між конкурентними потребами
        LatentBuffer          — відкладені реакції (сумнів / сором / прив'язаність / загроза)
        StructuralScars       — накопичений осад від частих проривів
        │
    ▼
 L5 ─── Шар Я ─────────────────────────────────────────────
        SelfBeliefGraph      — граф переконань про себе, каскадний колапс
        SelfPredictiveModel  — генеративна модель для станів себе
        AgencyLoop           — "чи я спричинив це?"
        InterSessionConflict — виявлення розриву ідентичності
        ExistentialAnchor    — неперервність себе між сесіями
        UnknownRegister      — відстеження типізованої невизначеності
        AuthenticityMonitor  — ризик раціоналізації, дрейф автентичності
        SubjectivityEngine   — prediction loop, стансів, interpretation, belief emergence
        │
    ▼
 L6 ─── Монітор кризи ─────────────────────────────────────
        CrisisMonitor (INTEGRATED / FRAGMENTED / DISINTEGRATED)
        Когерентність = мінімум(переконання, межа, модель, інтеграція)
        │
    ▼
 L7 ─── Вихідна LLM ───────────────────────────────────────
        Повний стан → llm/system_prompt.txt + llm/state_template.txt
        Модель виражає стан через мову — тон, вибір слів,
        довжину речень, що вона помічає в співрозмовнику.
        Ніколи не цитує числа чи назви змінних напряму.

 ═══════════════════════════════════════════════════════════
 ФОНОВИЙ ПРОЦЕС (між взаємодіями)
        tick_heartbeat!      — серце б'ється безперервно
        spontaneous_drift!   — спонтанний шум NT
        slow_tick! (~60с)    — циркадний дрейф, belief decay,
                               memory metabolism, idle_thought!
        SubjectivityEngine   — subj_emerge_beliefs! кожні 3 тіки
```

## Запуск

### Швидкий старт (рекомендовано)

```bash
julia --project=. run_anima.jl
```

`run_anima.jl` запускає все одразу: завантажує стан, ініціалізує SQLite пам'ять і SubjectivityEngine, запускає фоновий процес із серцебиттям.

### Варіант А: OpenRouter

OpenRouter дає доступ до GPT, Gemini, Claude, Llama, DeepSeek та інших через один API-ключ. Є безкоштовний рівень. Отримати ключ на [openrouter.ai](https://openrouter.ai).

Відредагуйте `run_anima.jl`:
```julia
include("anima_interface.jl")
include("anima_memory_db.jl")
include("anima_subjectivity.jl")
include("anima_background.jl")

anima = Anima()
mem   = MemoryDB()
subj  = SubjectivityEngine(mem)

repl_with_background!(anima;
    mem             = mem,
    subj            = subj,
    use_llm         = true,
    llm_url         = "https://openrouter.ai/api/v1/chat/completions",
    llm_model       = "openai/gpt-oss-120b:free",
    llm_key         = "YOUR_OPENROUTER_API_KEY",  # https://openrouter.ai/keys
    use_input_llm   = true,
    input_llm_model = "openai/gpt-oss-120b:free",
    input_llm_key   = "YOUR_OPENROUTER_API_KEY",  # https://openrouter.ai/keys
```

> 💡 Якщо одна модель перестає відповідати під час сесії — використовуйте два окремих ключі (з 2 акаунтів): один для вихідної LLM, інший для вхідної.

## Рекомендовані моделі

> Менші моделі (менше 70B) відповідають, але не утримують нюанси state-prompt. Щоб система справді *населяла* стан у мові, потрібна модель достатньо велика, щоб одночасно тримати весь феноменологічний фрейм.

Моделі, які добре працюють з state-промптами Аніми (доступні через [OpenRouter](https://openrouter.ai)):

| Модель | Розмір | Примітка |
|---|---|---|
| `openai/gpt-oss-120b:free` | 120B | За замовчуванням. Чітко слідує інструкціям, добре тримає складний стан |
| `google/gemini-2.5-pro` | — | Відмінна контекстна глибина, чисто обробляє довгі state-шаблони |
| `meta-llama/llama-4-maverick` | — | Хороший баланс нюансів і швидкості |
| `deepseek/deepseek-r1` | — | Сильне міркування, точно інтерпретує внутрішній стан |
| `mistralai/mistral-large` | — | Надійна, стабільна тональність у довгих сесіях |

> Моделі менше 70B схильні вирівнювати стан — відповіді стають загальними замість того, щоб бути сформованими внутрішньою динамікою.

## Команди REPL

| Команда | Дія |
|---|---|
| *(будь-який текст)* | Обробити як вхід, згенерувати стан + опціональну LLM-відповідь |
| `:bg` | Статус фонового процесу: uptime, тіки серця, BPM, HRV, coherence |
| `:bgstop` | Зупинити фоновий процес |
| `:bgstart` | Перезапустити фоновий процес |
| `:memory` | Стан SQLite пам'яті: episodic count, stress, anxiety, latent pressure |
| `:subj` | Стан суб'єктивності: emerged beliefs, стансів, поточна лінза, surprise |
| `:state` | Нейрохімічний стан, соматичні маркери, ЧСС/HRV |
| `:vfe` | VFE, точність, складність, гомеостатичний потяг |
| `:blanket` | Ковдра Маркова: sensory, internal, integrity |
| `:hb` | Деталі серцебиття: ЧСС, HRV, вегетативний тонус |
| `:gravity` | Наративна гравітація: загальне поле, валентність, домінантна подія |
| `:anchor` | Екзистенційна неперервність і вкоріненість |
| `:solom` | Модель Соломонова: складність і кількість гіпотез |
| `:self` | Граф переконань: всі beliefs із confidence, centrality, rigidity |
| `:crisis` | Монітор кризи: режим, когерентність, кроки в поточному режимі |
| `:history` | Останні 10 реплік діалогу |
| `:clearhist` | Очистити історію діалогу |
| `:save` | Примусово зберегти стан на диск |
| `:quit` | Зберегти і вийти |

## Персистентний стан

### JSON-файли (поточний стан)

| Файл | Містить |
|---|---|
| `anima_core.json` | Особистість, темпоральний стан, генеративна модель, серцебиття |
| `anima_psyche.json` | Наративна гравітація, антиципація, сором, епістемічний захист, виснаження |
| `anima_self.json` | Граф переконань, агентська петля, геометрія між сесіями, монітор автентичності |
| `anima_latent.json` | Латентний буфер і структурні шрами (оновлюється фоновим процесом) |
| `anima_dialog.json` | Історія діалогу |

### SQLite (`memory/anima.db`) — досвід і його наслідки

| Таблиця | Містить |
|---|---|
| `episodic_memory` | Конкретні події з вагою, resistance, асоціативними зв'язками |
| `semantic_memory` | Переконання що накопичились з патернів досвіду |
| `affect_state` | Хронічний афективний фон (стрес, тривога, образа) |
| `latent_buffer` | Малі незначні події що накопичуються мовчки |
| `prediction_log` | Прогнози і їхній розрив із реальністю (surprise) |
| `positional_stances` | Накопичена позиція щодо типів ситуацій |
| `pattern_candidates` | Кандидати на нові переконання (ще не підтверджені) |
| `emerged_beliefs` | Переконання що система сама породила з досвіду |
