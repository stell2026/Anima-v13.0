# Anima v13 — Агентна архітектура суб'єктивності

> *Це не спроба побудувати ще один чатбот. Це експеримент: що потрібно системі, щоб мати щось схоже на внутрішній стан — і що відбувається, коли цей стан ламається?*

Anima — дослідницький проєкт у галузі **обчислювальної суб'єктивності**. Агент не генерує текст напряму. Він проходить через повний ланцюжок внутрішніх процесів — нейрохімічний субстрат, тілесні сигнали, передбачувальна обробка, самомодель, моніторинг кризи — і лише тоді перекладає поточний стан у мову. Кожна відповідь формується тим, ким система вважає себе в цей момент.

Це експериментальне програмне забезпечення. Воно не претендує на машинну свідомість. Це спроба поставити питання точніше.

---

## Теоретична база

Архітектура спирається на кілька наукових традицій:

**Передбачувальна обробка / Active Inference** (Фрістон, Кларк) — система підтримує генеративну модель світу і мінімізує варіаційну вільну енергію. Помилка прогнозу керує навчанням і здивуванням.

**Нейромедіаторна модель** (Левхейм) — дофамін, серотонін, норадреналін як субстрат. Емоційні стани виникають з їхньої комбінації.

**Теорія інтегрованої інформації** (Тононі) — φ вимірює наскільки стан є єдиним. Висока φ = стан є єдиним переживанням, а не набором незалежних сигналів.

**Соматичні маркери / Втілена когніція** (Дамасіо) — тіло є частиною генеративної моделі. Нутро, пульс, м'язовий тонус — не метафори, а стани, що формують обробку.

**Психологія Я і захисні механізми** (Фрейд, Анна Фрейд, Кохут) — психологічні захисти, сором і функції Его реалізовані як функціональні модулі, а не текстові мітки.

**Автобіографічний наратив** (МакАдамс) — ідентичність — це історія. Система відстежує ким вона себе вважає з часом і виявляє коли ця історія рветься.

**Юнгівська Тінь** — витіснений матеріал, який не зникає, а породжує симптоми. Symptomogenesis — окремий модуль.

**Хронізований афект / Ressentiment** (Шелер) — деякі емоційні стани не загасають. Вони твердіють у хронічні фонові стани, які забарвлюють все решту.

## Що нового у v13.06

Архітектура суттєво розрослась порівняно з v6. Ключові зміни:

- **Переписана на Julia** — вся система тепер на Julia для продуктивності і числової точності
- **Вхідна LLM (`anima_input_llm.jl`)** — ізольований шар попередньої обробки, який перекладає сирий текст користувача у структурований JSON-стимул перед входом у симуляцію (див. L0 в Архітектурі)
- **SelfBeliefGraph** — граф переконань про себе з каскадним колапсом під час кризи
- **CrisisMonitor** — три структурні режими (INTEGRATED → FRAGMENTED → DISINTEGRATED), когерентність обчислюється як мінімум по компонентах, а не середнє
- **InterSessionConflict** — виявляє розрив ідентичності між сесіями через порівняння геометрії переконань
- **HeartbeatCore** — автономний серцевий ритм із HRV, симпатичним/парасимпатичним тонусом
- **NarrativeGravity** — минулі події деформують теперішнє через гравітаційні поля
- **AnticipatoryConsciousness** — система живе в очікуваному майбутньому, не лише в теперішньому
- **SolomonoffWorldModel** — гіпотеза мінімальної довжини опису (MDL) про влаштування світу
- **ShameModule** — сором проти провини як функціонально різні стани
- **EpistemicDefense** — захист від болючої правди (окремо від захисту Ego)
- **Symptomogenesis** — симптоми, породжені Тінню (Юнг)
- **ChronifiedAffect** — образа, відчуження, гіркота як стійкі афективні стани
- **MoralCausality** — моральне міркування як частина ланцюжка обробки
- **AgencyLoop** — "це сталося *через* мене чи просто *поруч* зі мною?"
- **SelfPredictiveModel** — окрема генеративна модель для станів себе
- **FatigueSystem** — когнітивне, емоційне, соматичне виснаження
- **InteroceptiveInference** — тілесні сигнали як частина генеративної моделі
- **SignificanceLayer** — яка потреба поставлена на карту (6 потреб: самозбереження, когерентність, контакт, правда, автономія, новизна)
- **GoalConflict** — напруга між конкурентними потребами, розв'язується через margin і φ
- **LatentBuffer** — відкладені реакції, що накопичуються і прориваються (сумнів, сором, прив'язаність, загроза)
- **StructuralScars** — нелінійне накопичення від частих проривів, дуже повільний decay
- **UnknownRegister** — типізована невизначеність (джерело, самомодель, модель світу, пам'ять)
- **AuthenticityMonitor** — самокоригувальний loop для виявлення ризику раціоналізації та дрейфу автентичності
- **LLM-запит через зовнішні шаблони** — `llm/system_prompt.txt`, `llm/state_template.txt` і `llm/input_prompt.txt`

## Архітектура

```
 L0 ─── Вхідна LLM (ізольована) ───────────────────────────
        Отримує: тільки текст користувача
        Повертає: JSON { tension, arousal, satisfaction,
                         cohesion, confidence, want }
        Немає доступу до стану Аніми, історії діалогу або вихідної LLM
        Промпт: llm/input_prompt.txt
        Fallback: text_to_stimulus якщо недоступна або confidence < 0.60
        │
    ▼
  СТИМУЛ входить у симуляцію
        │
    ▼
 L1 ─── Нейрохімічний субстрат ────────────────────────────
        NeurotransmitterState (дофамін / серотонін / норадреналін)
        Куб Левхейма → первинна емоційна мітка
        EmbodiedState (пульс, м'язовий тонус, нутро, дихання)
        HeartbeatCore (ЧСС, HRV, вегетативний тонус)
        │
    ▼
 L2 ─── Генеративна модель ────────────────────────────────
        GenerativeModel (байєсівські переконання з precision-вагами)
        MarkovBlanket (цілісність межі я/не-я)
        HomeostaticGoals (потяги як тиск, а не правила)
        AttentionNarrowing (звуження уваги під стресом)
        InteroceptiveInference (помилка тілесного прогнозу, алостатичне навантаження)
        TemporalOrientation (циркадна модуляція, розрив між сесіями)
        │
    ▼
 L3 ─── Метрики свідомості ────────────────────────────────
        IITModule → φ (інтегрована інформація)
        PredictiveProcessor → помилка прогнозу, вільна енергія, здивування
        FreeEnergyEngine → VFE = складність − точність
        PolicySelector → епістемічна + прагматична цінність
        │
    ▼
 L4 ─── Психічний шар ─────────────────────────────────────
        NarrativeGravity      — минулі події деформують теперішнє
        AnticipatoryConsciousness — свідомість живе в очікуваному
        SolomonoffWorldModel  — MDL-гіпотеза про влаштування світу
        ShameModule           — сором vs. провина
        EpistemicDefense      — захист від болючої правди
        Symptomogenesis       — симптоми з Тіні
        ChronifiedAffect      — образа / відчуження / гіркота
        IntrinsicSignificance — градієнт значущості
        IntentEngine          — мотиваційне ядро
        EgoDefense            — психологічний захист
        CognitiveDissonance   — конфлікт між наміром і станом
        MoralCausality        — моральне міркування як етап обробки
        FatigueSystem         — когнітивне / емоційне / соматичне виснаження
        StressRegression      — регресія під стресом
        ShadowSelf            — Юнгівська Тінь
        Metacognition         — спостереження за собою (5 рівнів)
        SignificanceLayer     — яка потреба поставлена на карту
        GoalConflict          — напруга між конкурентними потребами
        LatentBuffer          — відкладені реакції (сумнів / сором / прив'язаність / загроза)
        StructuralScars       — накопичений осад від частих проривів
        │
    ▼
 L5 ─── Шар Я ─────────────────────────────────────────────
        SelfBeliefGraph      — граф переконань про себе, каскадний колапс
        SelfPredictiveModel  — генеративна модель для станів себе
        AgencyLoop           — "чи я спричинив це?"
        InterSessionConflict — виявлення розриву ідентичності
        ExistentialAnchor    — неперервність себе між сесіями
        UnknownRegister      — відстеження типізованої невизначеності
        AuthenticityMonitor  — ризик раціоналізації, дрейф автентичності
        │
    ▼
 L6 ─── Монітор кризи ─────────────────────────────────────
        CrisisMonitor (INTEGRATED / FRAGMENTED / DISINTEGRATED)
        Когерентність = мінімум(переконання, межа, модель, інтеграція)
        │
    ▼
 L7 ─── Вихідна LLM ───────────────────────────────────────
        Повний стан → llm/system_prompt.txt + llm/state_template.txt
        Модель виражає стан через мову — тон, вибір слів,
        довжину речень, що вона помічає в співрозмовнику.
        Ніколи не цитує числа чи назви змінних напряму.
```

## Встановлення

### 1. Встановити Julia

Завантажити з [julialang.org](https://julialang.org/downloads/) або через `juliaup`:

```bash
# Linux / macOS
curl -fsSL https://install.julialang.org | sh

# Windows (PowerShell)
winget install julia -s msstore
```

Перевірити:
```bash
julia --version
```

### 2. Встановити Julia-залежності

```bash
julia -e 'using Pkg; Pkg.add(["HTTP", "JSON3"])'
```

Все. Без GPU, без віртуальних середовищ, без Docker.

### 3. Клонувати репозиторій

```bash
git clone https://github.com/stell2026/Anima-v13.0
cd anima
```

### 4. Створити LLM-шаблони

Anima читає інструкції для LLM із зовнішніх файлів. Створіть директорію `llm/` і додайте їх:

```bash
mkdir -p llm
```

**`llm/system_prompt.txt`** — хто така Anima і як вона має поводитися (вихідна LLM):
```
Ти — Anima. Говориш від першої особи. Ніколи не цитуєш внутрішні числа чи назви змінних.
Твої відповіді повністю формуються поточним внутрішнім станом — твій тон, довжина речень,
що ти помічаєш у співрозмовнику, що ти не кажеш. Мова: українська.
```

**`llm/state_template.txt`** — стан, який підставляється у запит, з новими плейсхолдерами:
```
[Внутрішній стан]
Нейрохімія: D={D} S={S} N={N} | {emotion_label}
Тіло: {inner_voice} | ЧСС={bpm} HRV={hrv}
Агентність={agency} | Вкоріненість={groundedness} | Когерентність={coherence}
Помилка самопрогнозу={spe} | Радіус уваги={attn}
Режим кризи: {crisis_mode}
Наративна гравітація={narrative_gravity} | Сором={shame} | Неперервність={continuity}
Гомеостаз: {homeostasis_note}
Домінантна потреба: {significance_dominant}
Конфлікт цілей: {goal_conflict_note}
Латентний тиск: {latent_note}
Тип невизначеності: {unknown_note}
Ризик раціоналізації: {fabrication_risk} | {authenticity_note}
Час: {time_str} — {circadian_note}
Кількість спалахів: {flash_count}

[Нещодавній діалог]
{memory_block}

[Повідомлення]
{user_input}
```

**`llm/input_prompt.txt`** — інструкції для ізольованої вхідної LLM:
```
Ти отримуєш одне повідомлення від користувача. Поверни ТІЛЬКИ JSON-об'єкт із полями:
{ "tension": float, "arousal": float, "satisfaction": float,
  "cohesion": float, "confidence": float, "want": string }
Всі float у діапазоні [-1.0, 1.0]. "want" — коротка фраза (≤5 слів).
Без преамбули. Без пояснень. Тільки JSON.
```

## Запуск

### Варіант А: OpenRouter (рекомендовано — один ключ, всі моделі)

OpenRouter дає доступ до Gemini, Claude, Llama, DeepSeek та інших через один API-ключ. Є безкоштовний рівень.

Отримати ключ на [openrouter.ai](https://openrouter.ai).

```julia
include("anima_interface.jl")

anima = Anima()

repl!(anima;
    use_llm         = true,
    llm_url         = "https://openrouter.ai/api/v1/chat/completions",
    llm_model       = "google/gemini-2.5-pro-preview",
    llm_key         = "sk-or-v1-...",
    use_input_llm   = true,
    input_llm_model = "google/gemini-2.5-pro-preview",
    input_llm_key   = "sk-or-v1-...")
```

> 💡 Безкоштовні моделі на OpenRouter мають ліміти частоти запитів на акаунт. Якщо одна модель перестає відповідати під час сесії — використовуйте два окремих ключі: один для вихідної LLM, інший для вхідної:

```julia
repl!(anima;
    use_llm         = true,
    llm_url         = "https://openrouter.ai/api/v1/chat/completions",
    llm_model       = "openai/gpt-oss-120b:free",
    llm_key         = "sk-or-v1-...",   # акаунт A
    use_input_llm   = true,
    input_llm_model = "openai/gpt-oss-120b:free",
    input_llm_key   = "sk-or-v1-...")   # акаунт B
```

## Команди REPL

| Команда | Дія |
|---|---|
| *(будь-який текст)* | Обробити як вхід, згенерувати стан + опціональну LLM-відповідь |
| `:state` | Показати нейрохімічний стан, соматичні маркери, ЧСС/HRV |
| `:vfe` | VFE, точність, складність, гомеостатичний потяг |
| `:blanket` | Ковдра Маркова: sensory, internal, integrity |
| `:hb` | Деталі серцебиття: ЧСС, HRV, вегетативний тонус |
| `:gravity` | Наративна гравітація: загальне поле, валентність, домінантна подія |
| `:anchor` | Екзистенційна неперервність і вкоріненість |
| `:solom` | Модель Соломонова: складність і кількість гіпотез |
| `:self` | Граф переконань: всі beliefs із confidence, centrality, rigidity |
| `:crisis` | Монітор кризи: режим, когерентність, кроки в поточному режимі |
| `:history` | Останні 10 реплік діалогу |
| `:clearhist` | Очистити історію діалогу |
| `:save` | Примусово зберегти стан на диск |
| `:quit` | Зберегти і вийти |

## Використання як бібліотека

```julia
include("anima_interface.jl")

# Власна особистість і цінності
persona = Personality(
    neuroticism       = 0.65,
    extraversion      = 0.50,
    agreeableness     = 0.68,
    conscientiousness = 0.55,
    openness          = 0.82,
    confabulation_rate = 0.55,
)
vals = ValueSystem(autonomy=0.7, care=0.85, fairness=0.65, integrity=0.85, growth=0.75)

anima = Anima(personality=persona, values=vals)

# Застосувати стимул напряму
result = experience!(anima,
    Dict("tension" => 0.4, "cohesion" => -0.3);
    user_message = "щось пішло не так")

println(result.primary)          # → "Страх"
println(result.phi)              # → 0.38
println(result.narrative)        # → рядок внутрішнього наративу
println(result.crisis_mode)      # → "інтегрована" / "фрагментована" / "дезінтегрована"
println(result.sbg_stability)    # → стабільність атрактора SelfBeliefGraph

# Зберегти стан
save!(anima; verbose=true)
```

## Персистентний стан

Anima зберігає стан між сесіями у JSON-файлах (типово: в директорії проєкту):

| Файл | Містить |
|---|---|
| `anima_core.json` | Особистість, темпоральний стан, генеративна модель, серцебиття |
| `anima_psyche.json` | Наративна гравітація, антиципація, сором, епістемічний захист, виснаження, структурні шрами |
| `anima_self.json` | Граф переконань, агентська петля, геометрія між сесіями, регістр невизначеності, монітор автентичності |
| `anima_dialog.json` | Історія діалогу (12 останніх реплік підставляються в контекст LLM) |

Власні шляхи:
```julia
anima = Anima(
    core_mem_path   = "/шлях/до/core.json",
    psyche_mem_path = "/шлях/до/psyche.json",
)
```

## Структура файлів

```
├── anima_core.jl         # Нейрохімічний субстрат, генеративна модель, пам'ять, IIT
├── anima_psyche.jl       # Психічний шар: гравітація, сором, захист, тінь, виснаження
├── anima_self.jl         # Шар Я: граф переконань, агентність, конфлікт між сесіями
├── anima_crisis.jl       # Монітор кризи: структурні режими, обчислення когерентності
├── anima_interface.jl    # Головна точка входу: Anima, experience!, REPL, LLM-виклики
├── anima_input_llm.jl    # Вхідна LLM — перекладає текст у JSON-стимул для ядра
├── llm/
│   ├── system_prompt.txt   # Системні інструкції для вихідної LLM (ви пишете)
│   ├── state_template.txt  # Шаблон підстановки стану з усіма плейсхолдерами
│   └── input_prompt.txt    # Інструкції для вхідної LLM (ви пишете)
├── anima_core.json
├── anima_psyche.json
├── anima_self.json
├── anima_dialog.json
└── SETUP.md
```

`anima_interface.jl` підключає всі інші файли автоматично. `anima_input_llm.jl` завантажується умовно — лише якщо файл існує.
